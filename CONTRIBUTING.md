# Contributing to DrugGuard

Thank you for your interest in contributing to DrugGuard! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/diabetic-ddi.git`
3. Create a branch: `git checkout -b feature/your-feature-name`

## Development Setup

Follow the installation instructions in the main README.md file.

## Code Style

### Python (Backend)
- Follow PEP 8 style guide
- Use type hints where possible
- Maximum line length: 100 characters
- Use meaningful variable names

### JavaScript/React (Frontend)
- Use functional components with hooks
- Follow ESLint rules
- Use meaningful component and variable names

## Commit Messages

Write clear, descriptive commit messages:
```
feat: Add eGFR-based dose adjustment calculator
fix: Resolve OCR false positives for generic drug names
docs: Update API documentation for diabetic endpoints
```

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Test your changes thoroughly
3. Update documentation if needed
4. Create a pull request with a clear description
5. Respond to any review feedback

## Areas for Contribution

- Additional clinical rules and guidelines
- ML model improvements
- UI/UX enhancements
- Documentation improvements
- Bug fixes
- Performance optimizations

## Questions?

Feel free to open an issue for questions or discussions.
